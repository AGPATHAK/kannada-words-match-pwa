# Issue M4-01 — Define the 51–100 curated word set and supporting schema decisions

## Goal

Define the second 50-word curated set (words 51–100) and lock the content-structure decisions needed before implementation.

## Why

The app has reached the point where the initial 50-word deck is becoming too small for meaningful ongoing use. Before integrating more content, the next 50 words should be curated deliberately so the app remains a trainer, not a random vocabulary game.

## Scope included

- define the 51–100 word set as a reviewable content artifact
- keep the new set focused on high-frequency everyday vocabulary
- keep the expansion mostly noun-based, with only a few very high-utility verbs if they remain unambiguous
- preserve or extend the current schema fields as needed
- define stage placement for the new words
- document any content-structure decisions needed before integration

## Out of scope

- no app-code changes yet
- no personal deck
- no Devanagari display-mode work
- no PWA packaging
- no major schema redesign

## Acceptance criteria

- the next 50 words are defined in a clean reviewable artifact
- the expanded 100-word content philosophy is explicit
- stage placement is clear
- schema compatibility with the app is preserved or minimally refined

## Validation

- review the 51–100 list for ambiguity and usefulness
- confirm the list remains coherent with the original 1–50 set
- confirm the schema remains practical for the app

## Likely files

- deck/content markdown or JSON artifact
- supporting notes if needed

## Constraints

- prioritize usefulness and clarity over variety
- do not allow the next 50 to become an incoherent random mix
